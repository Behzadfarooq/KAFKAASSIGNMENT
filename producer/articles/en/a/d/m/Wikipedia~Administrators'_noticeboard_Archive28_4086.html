Usernames aren't "autoblocked" really; they are just disabled because the underlying IP is blocked. There is no actual block placed against the account; when the block of the IP is removed, the username will immediately unblock. IPs, however, actually have a physical autoblock set against them, which must be separately removed (because it is actually a separate block). The easiest way to solve this is for the blocked user to identify the IP address they are using (it is listed in the blocked user text they see when they try to edit) and have an admin physically unblock that IP (i.e., enter it directly into the unblock screen.)
