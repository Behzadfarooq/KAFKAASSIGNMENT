[edit] Finished in the Latest Release (v1.2.1)

[edit] Minor Changes

  1. VandalProof no longer requires popups to revert edits.
  2. It now also marks reversions as minor edits as this seems to be the standard for administrative rollbacks.
  3. The edit summary left when reverting an edit has changed. Formerly, it would state, "Reverted edits by AmiDaniel (talk) to last version by 209.189.130.115 using VandalProof." Now it would state, "Reverted edits by AmiDaniel (talk) to version 49491423 by 209.189.130.115 using VandalProof," where 49491423 is the oldid of the version to which you are reverting. I personally find this more descriptive and helpful, as VandalProof's rollback, unlike the sysop rollback, does not necessarily revert to the previous version by the previous author--it can revert as far back as the user wants it to. I would be open to considering a date in place of the oldid; however, I fear this will be quite difficult to implement (and it's a lot of info to fit into an edit summary, possibly too much for an edit summary to contain).
  4. When you prod or nominate an article for speedy deletion or a transwiki move, it will now automatically add the article to your watchlist.
  5. When posting a warning or other remark on a user's talk page, it will mark this edit as minor.
  6. Additional {{db-band}} menu option.
  7. Under Article Tools, there is now an option to view the Article's logs, which will load in a separate window (this will quickly show if the article has been deleted previously, etc.). There is also a Search Google option, which can aid in cases where a topic is actually notable but where the writing is poor, as well as in cases where an article is well-written but non-notable.
  8. Under User Tools there is now a Block Log option and a User Logs option (helpful with page move vandalism, etc.)
  9. I've now incorporated my raw edit counter into VP which is accessible under User Tools (for any user) and Moderator Tools (mods only).

[edit] Navigation tools

  1. VP now has a "Navigate" menu with the options Forward (Ctrl+W), Back (Ctrl+B), and To Address.
  2. It also has an address bar that keeps track of every page you view during one session (in other words, it keeps track of your history).

[edit] Recent Changes Filtering

There's now a small drop-down menu next to the "Update" button with the following options:

  1. Show/Hide edits by white-listed users
  2. Show/Hide edits by logged-in users
  3. Show/Hide bot edits
  4. Show/Hide my edits
  5. Show/Hide minor edits

[edit] Stub-Sorter

Not really much of a counter-vandalism tool, but it still comes in useful on RC patrol.

  1. Under the new Article Tools menu there is now a quick sort item that contains about 30 top-most stub types. Clicking one will sort the article you're viewing as the appropriate stub (and replace any other stubs on the article).
  2. When you add a stub type, it cleans up the whole category/stub mess much like AWB does. (Such as here)
  3. Clicking Sort Stub will then load a small form that displays the stubs already added, giving you the option to replace the stubs or append one. It also gives you the option to preview the change before saving it. Here you can manually type in the stub or:
  4. A tree will load in this dialog that categorically displays all of the stub types listed on Wikipedia:Stub types. Clicking on one will then replace the manual input box with the stub type you picked. (This is still a little ugly-looking; I'll clean it up sometime soon. Parsing the data from Wikipedia:Stub types into the tree was an absolute bitch, so I hope you like it! lol.)

[edit] Global Whitelist/Blacklist

Users may now merge their local whitelists and blacklists into a globalized list, and others may subsequently merge the entire list back into their global list. The list has been obfuscated so that only VandalProof users can edit it; however, this is still going to be primarily on the honor system. If we have too many problems with Jimbo being blacklisted, etc., I may have to disband this feature.

[edit] Custom Rollback

I've found myself often wanting to post different messages after reverting a change in place of my typical custom messages, and so I've now created an optional "Rollback (Custom)" button. To enable the custom rollback, you will have to edit your "CustomButtons.txt" file and replace one of the buttons with:

 <ROLLBACK 7>
 Rollback (Custom)
 ##CUSTOMWARNING

This will create a rollback button that, when you click it, will load a dialog box (I'm getting a pic for this; hold on) that will allow you to select any of the predefined templates or to input your own, and you can also choose either to append or not append User:AmiDaniel/Rv-ref to the end of the warning.

[edit] Expanded History View Option

It always frustrates me when I see an edit summary like "Reverted to 99999" and I have no clue what version that is without hovering over each link, so I've now created a menu item to expand the history. When viewing a history page, clicking on File/Expand history will append the oldid of each revision to the end of the date. It will also had a "diff" link before each item. (I'm getting an image of this as well; it should be up in a bit.)
