This FAQ is intended as a troubleshooting guide. It contains solutions to problems that previous users have had. Before reporting a bug, please check this FAQ. If you cannot find the answer to your problem here, you can ask about it on the discussion page, or in the VP forums. For more help, you could view the help page.

Parts of this FAQ may be incorrect or outdated for newer (or older) VP releases.

[edit] VandalProof 1.3.#

The following solutions may not be relevant for users of VandalProof 1.2.# releases.

[edit] "Run-time error '91'"

Problem

When running VandalProof, for no apparent reason an error message appears stating:

Run-time error '91': 
Object varible or With Block varible not set.

and VandalProof closes.

Solution

This bug is believed to have been fixed in the next release.

[edit] "You are not on the user list"

Problem

When running VandalProof, and you click "Verify Authorization", you receive a message that says either the userlist is corrupt, or you are not on it.

Solution

There are several possible causes for this. The moderator who approved you may have forgotten to add your name to the list, your name may have been lost when the userlists were merged, or it may be a bug. To fix it, you can:

Ensure your username is on the list here. If not, re-add it to the "Awaiting Approval" list.
Contact one of the VandalProof moderators and ask them to try and fix it for you.
There is a known bug with some usernames not being recognised. Unfortunately, there is no known fix for this problem.

[edit] Page loaded, but rollback buttons are greyed out

Problem

The page loads correctly when the recent change is selected, but the rollback buttons stay greyed out and cannot be used.

Solution

This is the double warning safeguard. If another user, or AntiVandalBot has already edited the page, you will not be able to rollback the changes.

[edit] VandalProof 1.2.#

The following solutions may not be relevant for users of VandalProof 1.3.# releases.

[edit] Update not Working

Problem

When I click the update buttons in the tabs, no information is retrieved.

Solution

Make sure that Enhanced Recent Changes is OFF.
To access this, go to 'my preferences', click the tab 'Recent Changes', and uncheck 'Enhanced Recent Changes'.

[edit] Automatic "Save page" not Working

Problem

Hitting {rollback} does not actually "rollback" the vandal's edits. (i.e. does not press the "Save page" button.).

Some other variations of this...
  1. will warn the vandal, but won't revert.
  2. no edit summary.

Solution

To fix this, you must have pop-ups enabled: place {{subst:navpop}} in your your personal monobook.js file and save the changes. You may need to clear your cache before the changes can be seen.

[edit] Download Fails in Mozilla FireFox

Problem

When using Mozilla Firefox 1.5.0.1, the download does not work (i.e. it brings up the Choose program to open with... dialogue box)

Condition

The default browser is NOT internet explorer. (i.e. "I browse the web with an alternate browser").

Solution

The file can still be downloaded, but should be renamed to include ".exe". With this done, the program will work fine.

[edit] Internet Explorer Script Error

Problem

I have removed and reinstalled, loaded it in safe mode, system restored and still it pops up every second or so. Please help?! - Glen T C 21:00, 11 April 2006 (UTC)

I believe this has to do with missing or outdated component files as I've explained above. I'm working on an install that actually *installs*, but it may be a while as the PaD Wizard I used for the first simply will not create a correct setup for this app, meaning I have to write my own setup. Please bear with me! I'll get this all sorted out. Thanks. AmiDaniel (Talk) 21:41, 11 April 2006 (UTC)
Not an ideal solution but I created a new user in windows (admin) then downloaded the install file - worked first time. Just dragged the txt files into a shared folder and voila! - Glen T C 00:08, 12 April 2006 (UTC)

Solution

Usually after a large number of edits with VandalProof, a script warning will appear:
----
Warning! An error has occurred with the scripts on this page.
ERROR INFORMATION:
Blah Blah Blah. Blah Blah Blah. Blah Blah Blah. Blah Blah Blah. Blah Blah Blah. Blah Blah Blah. Blah Blah Blah. Blah Blah Blah. Blah Blah Blah. Blah Blah Blah. Blah Blah Blah. Blah Blah Blah. Blah Blah Blah.
Do you wish to continue running scripts on this page?
Yes No
----
It occurs at least twice on every page, making it very annoying to use the tool. However, it can be fixed by clearing the browsing history and deleting temporary files.
How to Fix:
Instructions:
For Internet Explorer 6.x:
1) Go to Tools, then select Internet Options.
2) Under the General Tab, select the Settings button.
3) Click Delete Files.
4) Wait.
5) Wait.
6) Wait.
7) Voila! Gone!
For Internet Explorer 7 Beta 2 Preview:
1) Press Alt+T, then D
2) Select Delete Files under the Temporary Internet Files section.
3) Voila! Gone!
If this doesn't work, then the monobook.js page must be cleared out, and then only pop-ups can be installed.
Note: When clearing the browsing history, it will delete saved passwords. Therefore, you must re-login to Internet Explorer in order to use VandalProof (or else you will get a warning message which says that VandalProof could not log you in.)


Hopefully, the script error message could be avoided in the next version of VandalProof, as it took me 2 hours to figure out why it was giving me that message.


Primate#101 17:42, 25 April 2006 (UTC)

[edit] Rollback Functions Greyed Out

Problem

The rollback function is grey no matter what happens; when the page loads, it blinks to black for a split second, then is greyed out again. I do have popups.js equipped. _-M o P-_ 06:45, 25 April 2006 (UTC)

Solution

The monobook.js page must be cleared out, and then only pop-ups should be installed by placing {{subst:navpop}} in your monobook.js page.

[edit] "You may no longer use this version..." Error Message

Problem

On starting VandalProof, the error message:

You may no longer use this version, please download the newest version.

is displayed, despite the fact that the program was just downloaded.

Solution

The Setup.exe file may not contain the latest version. You should download a new executable file from here and place it in a new folder. You should also update your shortcuts to refer to this file, rather than the other one. Do not download the .ZIP version of the file.
