The tone mark is added to the vowel in the syllable that comes first in this sequence: a o e i u ü. For example, ai takes the tone mark on a, and ia also takes the tone mark on a. The only exception is iu which takes the tone mark on u.

Semantic algorithm on how to put up wikitables


start with

{| class="wikitable"
|-

count the number of total rows and total columns of your table. keep always in mind that the number of columns for each line has to add up with your total number of columns. when adding up, count both the actual entries made with "| string || string2 || string3" (that would make three columns) AND the number of spanning fields made with "| colspan="5" | string" (that would make five columns ... so altogether this makes eight columns.

identify the upperleft field. look at how many y = rows and x = columns it spans if x or y is different from 1 than type

| rowspan="y" colspan="x" | string

where string is the string you want to place in this field. if the first row has more fields, than input

| secondfield || thirdfield || fourthfield

note that this row is a child of your field "string"

to start the next row, type

|-

note that all following lines will be row children of your "string" field until you have reached a number of y rows. after this a number of x columns will start which you have been specifing with rowspan="x"

go on for so until you finished all your rows. note that while you do not need to specify how many rows your table has, the number of columns always has to be the same for each row. this is due the fact that wikitable's syntax works by first demanding a row, than by specifing this line's columns.

end your table with

|}

if you want to highlight a field, put "!" instead of "|" at the beginning of this fields line. to "unlight" the following fields seperate them from the highlighted with a newline. rember that for the atomic field size, you don't need to specify colspan and rowspan.

For more help, see Help:Table
