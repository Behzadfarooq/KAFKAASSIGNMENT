The article was using an old style of reference. I changed it to a newer style, where you embed the reference between a pair of html tags, the <ref></ref> pair. Then, at the end of the article, you put a <references/> tag. That prints out the references, numbered, in order. If there are letters following the reference number, each one is a link to one of multiple references to that link.
I give each instance of a <ref></ref> pair a name, even if I only use that reference once. And when I use a reference multiple times I paste the whole reference. Other people will put something like <ref name=ThisIsAName/> for the subsequent references. I don't do this because, at some point, another editor might erase the paragraph with the first reference, or they might move it to a place at the document. I figure I prevent errors if I make every reference a complete reference. Don't hestitate to leave me a message on my talk page if you have any questions.
Can I ask if you have considered creating a userid? Now I sign my message by putting four tildes in a row, like this: ~~~~ That gets translated into a signature and time stamp. You can do that, even though you don't have a userid. -- Geo Swan 02:48, 10 July 2006 (UTC)
