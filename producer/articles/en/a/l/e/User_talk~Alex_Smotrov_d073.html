[edit] QPreview with wikEd

Any chance of making QPreview compatible with wikEd? The QPreview button doesn't show up in the wikEd toolbar, and even when I choose to show the normal toolbar (which I usually don't), the QPreview button doesn't work. It's definitely because of wikEd; I turned off wikEd and it worked just fine. It would really help the script if you could get it to work while I'm using wikEd. Thanks, Pyrospirit Flames Fire 17:50, 28 March 2007 (UTC)

Well, I don't want to make QPreview check for wikEd presence... so I came up with this 'compatibility patch' which adds QPreview button as wikEd custom button.
//define wikEd custom button and custom toolbar for QPreview
var wikEdButton = [];
wikEdButton[100] = ['wikEdQPreview', 'wikEdButton', 'Quick Preview', '', '16', '16', 'QPreview', 'javascript:WikEdEditButton(this, this.id, null, launchQPreview);' ];
var wikEdButtonBar = [];
wikEdButtonBar['custom1'] = ['wikEdButtonBarCustom1', 'wikEdButtonBarCustom1', 'wikEdButtonsCustom1',  'wikEdButtonsCustom1', 44, 'My custom buttons', [100] ];
function launchQPreview(obj) {
 if (wikEdUseWikEd) WikEdUpdateTextarea();
 qPreview();
 //this 'hack' is needed so the button doesn't eat 1 symbol on the left; shame on Cacycle
 WikEdGetText(obj, 'selection, cursor');
 if (obj.selection.plain != '')  obj.changed = obj.selection
 else  obj.changed = obj.cursor;
 obj.changed.plain = null;
 obj.changed.keepSel = true;
}
You should add it into your monobook.js so it runs before wikEd is imported.
However this new button does not change to 'Wait...' when clicked and also assigned accesskey doesn't work. Let me know if you want these to be fixed as well.
By the way, I think it would be a good idea if you suggested Cacycle to incorporate QPreview functionality (only about 20 lines of code) into wikEd. — Alex Smotrov 22:31, 28 March 2007 (UTC)
Thanks, I'll use this for now and suggest it to Cacycle. Pyrospirit Flames Fire 23:17, 28 March 2007 (UTC)
