Application Virtualization is a field within computer science which deals with breaking the age old bond between physical hardware, operating system, and the program which runs on top of them. From the time the ENIAC first went into production in the mid nineteen forties, until the present day, software has been "installed" on the specific machine upon which it will execute.

[edit] Definition and overview

There are several qualities which must be inherent in an environment in order for it to be considered true application virtualization, rather than the much simpler approach of "application management". First, the runtime environment must completely isolate the application from both the hardware resources as well as any programming API's of the underlying operating system. By meeting this requirement, the encapsulated process would be capable of executing on any machine across a heterogeneous network. Secondly the environment needs to be able to dynamically assemble applications in real-time. In order to accomplish this goal a dispatching component would need to relocate code binaries in conjunction with facilities to handle load balancing. These facilities could utilize either algorithmic means, production rules, or a combination thereof, to decide which available machine on the network would most efficiently execute the task. In addition to dispatching an executable, there needs to be a reusable component object model from which applications can be dynamically assembled and reassembled. Lastly the environment must support stateful process migration. This involves the actual movement of a running process from one machine to another. Traditional disaster recovery methodologies use a technique called "cold failover", whereby an application would be shut down in one data center, and then be restarted in a secondary location. Process migration on the other hand stops the running application in midstream, creates a snapshot of the program data, relocates the executables along with member data, and then restarts the task. Newer, second generation application virtualization environments rely upon a more advanced technique, which is generically known as strong thread migration. This approach allows a process to continue to execute where it left off upon its arrival on a machine to which it is being migrated. One patented implementation, invented by Marc Schneiderman, is called automatic thread migration, and allows a tertiary process to maintain both the execution stack trace and the state of program data, bundling both of them along with Java bytecodes when migration to a new host takes place.

[edit] Advantages

Some advantages that application virtualization has over conventional distributed computing environments:

  • Faster delivery of higher quality applications.
  • Lower total cost of software ownership
  • Reduction in the number of servers needed.
  • Increased system reliability
  • Geometric scalability
  • Better performance
  • Reduced administration, deployment, monitoring, and maintenance costs

[edit] Preferred Uses For Application Virtualization

Common applications include:

  • Supply chain management
  • Electronic commerce
  • Financial applications
  • Insurance applications
  • Complex adaptive systems
  • Business types of business simulations
