Applejuice is a semi-centralized peer-to-peer file sharing network similar to the original eDonkey network.

[edit] Topology

The project offers server and client software for Applejuicenet. This means that the network is not managed by one company, organisation or server, but decentralized over many servers. The servers however, are central points for all users.

Users of the Applejuice client connect to an Applejuice server, search for and find files, also locate files with a given hash from links, and then download them directly from other clients (or "peers").

For locating resources, the Applejuice network uses a client-server system. However, the servers are also connected one to another, so if you are on a Server A you can locate files from users on the Server B (server-server). Clients also directly exchange information about servers and files.[citation needed]

The topology is comparable to IRC. Clients have to connect to a server, so it is somewhat centralized. However, there are many servers in each network, and there are also many networks. Client-client communication is similar to IRC's DCC.

[edit] Network

The servers don't store any files. All files are served by the clients. The servers index these files, and the clients exchange the files. The clients are actually Servents (P2P), regarding file sharing services. The used terms describe the role in the file searching services.

[edit] Credit System

The network is based on credits. For 1 byte upload you'll get 2 byte credits plus a 10% bonus of the Powerdownload ratio in addition, if the downloader activates the Powerdownload.

1 byte download cost 1 byte credits without Powerdownload. With activated Powerdownload you have to pay correspondingly more credits.

It must be pointed out:

  1. If you close the core all credits will reset to 0.
  2. If you have less than 0 credits you can't activate the Powerdownload and the download won't rise above 15 kbyte/sec. Furthermore the download won't be higher than your upload if you don't have credits.
  3. The Powerdownload stays disabled if you don't have the credits to pay for it.
  4. For incomplete chunk uploads the earned credits will be cancelled, because no one can use an incomplete chunk and it's a waste of bandwidth. A chunk can break, if the settings aren't configured correctly (e.g. the maximum upload in the AJ options is set much too high). If the chunk upload stops for more than 150 seconds a chunk will break.

[edit] Powerdownload

Clients which have enough credits can use the so called "Powerdownload". With activated Powerdownload you will get a better place in the upload queues from the other users and your download speed will rise (it works like an auction). You only have to "pay" for the next higher Powerdownload ranking, from the bidding of the second highest user. e. g. another user set his powerdownload ratio to 3,5 and you set the powerdownload ratio to 4,8 than you only pay 3,6 kbyte credits per 1 kbyte download (one step above the second highest bidding).

[edit] Structure of Client

The Client consists of 2 elements:

  • Core - the part which will do all operations, written in Java
  • GUI - a graphical user interface (C++,.NET, Java, PHP, Linux,..)

The elements are connected over a network (TCP/IP). The Core listens on a specific port, and the GUI connects and gets status information/sends requests.

The protocol for communication between core and GUI is open. Every application which implements it can be used as GUI. Currently, there are native Windows GUIs, an official Java GUI, a PHP-Webinterface and also a Linux-based GUI.

The core also runs without a connected GUI. Thus, you can close the GUI and go on swapping, or also shut down your local computer if the core runs on a server.

[edit] Usage

The Applejuicenet is used for content distribution with a given Hashlink. The system has also a built-in search to search servers for keywords, however this feature is only efficient if you're using the current snapshot core (version >=0.30.146.1203) in combination with the Java-GUI 0.70 CVS-Build. Furthermore you have to enable/forward the core-port in your firewall/router. The Java-GUI 0.70 CVS-Build requires the Java-Runtime-Environment 5.0 and has some smaller bugs, so you should use this GUI version only for search and the Java-GUI 0.61.2 for the rest (you can close the GUI, if you are longer away - the core can also up-/download without the GUI and it saves CPU-bandwidth). Certainly there are also Community Linksites exchanging clickable hashes and some information about the files.

[edit] Some tips at the beginning

  • You need to start the core before the GUI
  • Make an update from the core and the GUI after installation of the setup.exe package (in this package are older versions, but you need the setup.exe package because it contains the windows dll's, alternatively you could also copy the libraries separately from this site: http://www.applejuicenet.de/22.0.html?dir=lib ).
  • Use the latest 5.0 Java Runtime Environment
  • You should only install one version of the Java Runtime Environment at the same time, because multiple different versions can make trouble
  • You should download some widely spread and actual files at the beginning, because these files won't require the powerdownload usually
  • Open the core-port in the AJ-Options in your firewall and forward this port at your router, if you are using a router
  • add more servers at the register-card server, if you can't connect to a server with the button "get more servers here"
  • allocate more RAM to your core at the start, if you have a bandwidth >2 mbit. Normally the core will only get 64 mbyte by default - this allows your client to use ~57 active upload/download connections.
    For 3 mbit bandwidth I suggest to allocate 96 mbyte, for 6 mbit 128 mbyte (this allows ~119 active upload/download connections) and so on...
    You can allocate the RAM as follows:
    1. If you wish to start the core in the prompt type: java -Xmx128m -jar ajcore.jar
    2. If you would like to start the core over the Windows link: javaw.exe -Xmx128m -jar "C:\programs\appleJuice\core\ajcore.jar"
    Note: in the above example the RAM allocation is set to 128 mbyte, but you can also change the mbyte amount in the expression -Xmx128m with another number.
