The wiki servers used to have a script that ran on those servers. The script was named "tidy". The purpose of the script was to close any open HTML tags and remove any extraneous closed HTML tags. Today, the wikipedia server admins discovered that the "tidy" script was causing the servers to bog down and eventually crash. In order to improve server performance, the "tidy" script was taken offline.
The consequence of not having "tidy" around is that any improperly formatted HTML that would have gone unnoticed prior to this afternoon (since it would have automatically been cleaned up by the "tidy" program) is now quite visible and, in many cases, causing entire pages (or chunks thereof) to turn green or blue or be rendered in barely visible font sizes, and so on. People have been going around and correcting these errant HTML issues as they appear. Durova fixed the issue with the strikeout earlier, as shown in the edit summaries. → Ξxtreme Unction {yakłblah} 00:33, 15 November 2005 (UTC)
