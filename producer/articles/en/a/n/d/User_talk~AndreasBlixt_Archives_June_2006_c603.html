Ok, I'm not a native speaker, so that's very likely :) However the use of "can employ" instead of "employs" was intentional: often the template is used for code which is actually simple and doesn't employ any "esoteric" feature but suffers from unexpected browser bugs (often IE) and/or idiosyncrasies.
