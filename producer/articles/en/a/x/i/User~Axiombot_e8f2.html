What Axiombot is

Axiombot is an automated computer program based on patterns that catches vandalism. It is a set of check statements that if enough match true, it reverts. The patterns will be emailed upon request to users who can be trusted (for obvious reasons I'm not giving it to vandals)


What Axiombot is not

Axiombot is not an admin, it does not block users. Axiombot does not have the intelligence of a human being and it will not catch everything. Axiombot only issues warnings properly Axiombot does not catch all vandalism, only some obvious vandalism. RC Patrollers will always be needed.


Status Modes

Axiombot has two modes, angry and calm which can be set to either:

angry where it will revert as many times as it detects vandalism calm where it will revert only once and will not re-revert. The bot is normally calm and will only be set on angry in extreme cases


Safety Overrides

While Axiombot can do a lot of good, it can also do a lot of damage if it malfunctions. Therefore Axiombot has 2 major safety overrides:

Any member of the CVU with operator access can shutdown the bot by issuing "ctawkbot norevert" as an IRC operator. Admin blocking When in calm mode (its normal operation mode) Tawkerbot2 will only revert once and will not challenge a user.


Common errors that lead to an autorevert

Not formatting redirects properly, # REDIRECT is not #REDIRECT Redirected page name only the latter will work as a redirect, the former will trigger the bot. Not using a copyvio tag on a copyright violation. I have no idea how we can make that one work properly; if it happens just revert and delete the bot's warning. Server-based DB errors were causing some problems but that appears to be fixed now.
