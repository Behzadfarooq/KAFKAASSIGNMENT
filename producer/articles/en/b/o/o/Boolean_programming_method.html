In computer science the boolean programming method is a term used to describe a method of debugging, repairing, or bandaiding a problem in a program through excessive use of the variables of the Boolean datatype.

This method is often called a bandaid or a hack because it is a quick fix to a problem rather than actually fixing the origin of the problem. While the boolean programming method is effective and can be a programming time saver, it is not recommended because of the various efficiency and maintenance issues that ensue. The problem is not actually solved, just hidden, and often the computer has to do many extra comparison calculations in succession to deal with the exorbitant number of bools in the program. Also, all of these bools clutter and use up more stack space and physical memory.

This programming method was invented and perfected over time by Computer Engineering student Dominick O'Dierno while attending MSOE during the 2004-2005 school year. Its popularity is catching on with other students at the school, and as a result will play a prominent part in the programming portion of the senior design project being planned and implemented by O'Dierno.

An updated version of the Boolean Method is currently under development for the C# programming language. It was originally designed to be used with the C++ language, and modified to be used with Motorola 68HC11 assembly language during the 2005-2006 school year by O'Dierno and various other Computer Engineering students at MSOE.
