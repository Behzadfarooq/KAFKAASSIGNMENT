[edit] STBotD (STBot task 6, interwiki)

tasks • contribs • count • logs • page moves • block user • block log • flag log • flag bot

Operator: ST47

Automatic or Manually Assisted: Automatic, supervised at times

Programming Language(s): Python 2.5

Function Summary: Interwiki search and fixes on en only using -autonomous, -force, -hint:10(check for same name on 10 largest wikis), -continue

Edit period(s) (e.g. Continuous, daily, one time run): Daily, overnight, specific times can be set if you want me to, though it would probably be 12A - 10A GMT +/- 2 hours. I can run it only on weekends, or not on weekends, if you would like me to.

Edit rate requested: 2 edits per minute average

Already has a bot flag (Y/N): Not this account(want the contribs separate for east review)

Function Details: Uses pywikipedia's interwiki script to go page by page and search for missing or changed interwiki links. I'll review random edits afterwards to make sure it didn't remove anything it shouldn't have.

[edit] Discussion

I've gathered some diffs over time testing some settings at Special:Contributions/STBotD ST47Talk 15:32, 9 December 2006 (UTC)

What langauges? Where does it get the list of pages to check from? This will use a lot of requests and we already have several interwiki bots. It would be interesting if we could get some coordination among such bot operators (perhaps breakup tasks by letter ranges). Voice-of-All 20:19, 10 December 2006 (UTC)
It would obviously only be editing here, i'd haveto request approval elsewhere, I'd like to organize it so multiple operators are working together instead of individually, perhaps a pseudo-mini-wikiproject to organize and either crawl through categories or start at a and go down the list. As for load, the bot pauses for 5-6 seconds once in a while to reduce load. ST47Talk 21:02, 10 December 2006 (UTC)

Speedily Approved. Interwiki bot. Trial looks good as well. Just keep the edit rate within the requirements. -- RM 00:52, 12 December 2006 (UTC)
