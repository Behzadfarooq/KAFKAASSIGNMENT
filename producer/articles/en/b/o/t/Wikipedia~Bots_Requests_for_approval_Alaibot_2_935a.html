[edit] Alaibot

tasks • contribs • count • logs • page moves • block user • block log • flag log • flag bot

Operator: Alai

Automatic or Manually Assisted: Automatic

Programming Language(s): Python (replace.py from pywikipedia)

Function Summary: Parameterising {{importance}} by month

Edit period(s) (e.g. Continuous, daily, one time run): one time run to clear existing category; manually re-run as needed thereafter

Edit rate requested: 6 edits per minute

Already has a bot flag (Y/N): Y

Function Details: It's been proposed that Cat:Wikipedia articles with topics of unclear importance, which is getting quite large, be split up by month, on the pattern of the various maintenance resources in Cat:Wikipedia maintenance categories sorted by month. (Discussion here and here.) Doing this seems straightforward technically: a number of initial runs, based on database category timestamps, to split the current category (about 5,000 edits total) by adding a parameter to existing template usages. Thereafter on the order of up to about 2,000 per month at current rates of tagging (run probably roughly weekly, or more often (up to daily) if there's a backlog). Alai 16:24, 29 November 2006 (UTC)
