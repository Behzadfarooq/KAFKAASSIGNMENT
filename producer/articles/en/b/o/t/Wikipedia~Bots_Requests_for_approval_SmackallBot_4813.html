[edit] SmackallBot

tasks • contribs • count • logs • page moves • block user • block log • flag log • flag bot

Operator: codetiger

Automatic or Manually Assisted: Manually Assisted

Programming Language(s): Python

Function Summary: To revert recent changes if it contains some bad words in the list and to instruct the anonymous user to use a login to edit and welcome then for testing wiki. As this is just going to be tested on User:codetiger/bottestpage for sometime before the use of this bot is 100% tested.

Edit period(s) (e.g. Continuous, daily, one time run): daily

Edit rate requested: X edits per TIME I am sure this is going to be very slow.

Already has a bot flag (Y/N): N

Function Details: For now, I am testing this bot on my test page. This bot is expected to be up to revert changes by anonymouse users alone if they add words which are in the black list. And the users are instructed to use a login and the test template is added to their talk page.

[edit] Discussion

we already have such bots see TB2 and its clones. Betacommand (talk • contribs • Bot) 21:36, 27 November 2006 (UTC)
Is there room for another one of these bots? -- RM 15:21, 29 November 2006 (UTC)
what can this bot catch that the TBs can't? Betacommand (talk • contribs • Bot) 15:26, 29 November 2006 (UTC)
Well, more importantly, can it handle the cases where AVB gets a false positive and otherwise screws up. AVB2 is coming, sometime :) -- Tawker 21:13, 29 November 2006 (UTC)
Ok, so basically if we are going to consider this option, we need to know more specifically how it is going to work. -- RM 15:40, 30 November 2006 (UTC)
It sounds like this bot's teeth are too sharp, will it be blindly reverting and "warning" users who come to work on articles such as this one or this one? — xaosflux Talk 04:27, 1 December 2006 (UTC)
We already have User:AntiVandalBot that does general edit patrol (I don't what new stuff AVB2 does, other than be better), User:VoABot II that does move patrol, AOL, shared, and Host (open ports) IP edit checking, and can lock down pages from them, we have User:Shadowbot, which does general anti-spam work. I don't see any need for another such bot, especially starting from sratch. You may want to just tell those bot operators any suggestions/minor changes rather than make a new bot.Voice-of-All 23:09, 1 December 2006 (UTC)
Bot not approved, discussion archived. Voice-of-All 19:32, 10 December 2006 (UTC)
