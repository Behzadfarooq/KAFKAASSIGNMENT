Sometimes articles link to non-existent or malformed categories. If this can be taken care of just by fixing links in the articles (the link displays red) then the categories below can just be fixed and de-listed. If it requires actually deleting a category (the link shows blue) then it needs to be fixed and then listed on WP:CFD.

[edit] How to use these lists

Categories are "detected" in the below lists if they have weird, non-alphanumeric characters in the name. This may be an indication that either a syntax error was made on an article page (not unlikely if you only see one article for that category), or that the category should be renamed because it contains weird characters. If a category is listed here but does not have either of those problems, please add it to "Categories that appear to be ok". This will prevent it from being listed here in the future.

Categories should have two things: 1.) A description page, and 2.) Member articles and subcategories. A category might:

  • Have a description page but no articles or subcategories. (Blue, but not linked to from anywhere.) These should be listed on WP:CFD, or added to Category:Underpopulated categories if kept.
  • Have no description page but have member articles or subcategories. (Red, but linked to from other pages.) If the category is being kept, a description page should be added which specifies an appropriate parent category (like this: [[Category:NAME OF PARENT CATEGORY HERE]]). If the category is not being kept, the articles linking there should be moved to other categories, or at least removed from this one.
  • Have a description page and member articles or subcategories. (Blue, and linked from other pages.) This is the normal state of affairs. If a category like this needs to be renamed, please list it on Wikipedia:Categories for deletion (which also handles renaming).

Some category names appear black on the below lists. The script which produces automated lists does this because some category names that contain weird characters such that they cannot be linked to directly. You should still be able to reach these categories by visiting articles that link to them.

[edit] Character conventions

  • Namespace prefixes like "Wikipedia:" don't belong in category names. Use an alternative. For example, you can use "Wikipedia" as an adjective, without the colon.
  • Non-ASCII characters should be represented by their native UTF-8 (a Unicode representation) entity in category names. Neither HTML entities like & nor URL-encoded entities, like %2E should appear in category names as you see them in the wikitext when you edit a member article. You will see URL-encoded entities in URLs. (This, of course, is why it's easy to get confused and link to the wrong thing.)
  • Use "and" instead of "&" unless it is an integral part of a proper noun (e.g. "Law & Order").
  • To avoid the appearance of subpages, slashes should not appear in category names unless they are an integral part of a proper noun. (e.g. "OS/2") or unless there is another very good reason for having them. Try to rephrase or use hyphens instead (e.g. "Dallas-Fort Worth", "Minneapolis-St. Paul").
