[edit] Unicode display problems

Hi Baumfabrik, I saw your comments on Unicode display problems on the IPA page. Probably that page is not the best of places to direct such a question to, as the page is not really frequented by the tech gurus but more by us poor old linguists, and the problem affects really the whole of Wikipedia and not just this one page. Now, it seems there are two separate technical problems: that of character encoding, and that of fonts. As for character encoding, all pages seem to carry a correct "charset=utf-8" tag in their html header. That should really be sufficient for all modern browsers to select the proper Unicode encoding automatically without any user intervention. What browser are you using, and what settings did it have that made it not select UTF-8? -- The other question is fonts (that's what's behind your empty-boxes view.) The Wikipedia technique currently is that special characters be inclosed in a special tag, which gets translated into a html <span> class referencing a .css stylesheet. The stylesheets list the following font names. The idea is that if you have any one of these fonts from each of the three categories on your system, your browser will select it to display the characters in question. For IPA, I recommend "Doulos SIL" (freely available at www.sil.org). "IPA" is for phonetic characters, "polytonic" is for ancient Greek, and "Unicode" is for a wide range of other rare characters.

.IPA {
       font-family: Chrysanthi Unicode, Doulos SIL, Gentium, GentiumAlt, Code2000, TITUS Cyberbit Basic, DejaVu Sans, Bitstream Vera Sans, Bitstream Cyberbit, Arial Unicode MS, Lucida Sans Unicode, Hiragino Kaku Gothic Pro, Matrix Unicode;
       font-family /**/:inherit;
}
.Unicode {
       font-family: TITUS Cyberbit Basic, Code2000, Doulos SIL, Chrysanthi Unicode, Bitstream Cyberbit, Bitstream CyberBase, Bitstream Vera, Thryomanes, Gentium, GentiumAlt, Visual Geez Unicode, Lucida Grande, Arial Unicode MS, Microsoft Sans Serif, Lucida Sans Unicode;
       font-family /**/:inherit;
}
.polytonic {
       font-family: Athena, Gentium, Palatino Linotype, Arial Unicode MS, Lucida Sans Unicode, Lucida Grande, Code2000; 
       font-family /**/:inherit;
}

P.S. Oh, by the way, the warning message you suggested is actually already there, but at the end of the article. I agree it could possibly be enhanced. Lukas 18:55, 23 January 2006 (UTC)

[edit] Re: Geosynchronous

My understanding is that geostationary is a type (that is, a subset) of geosynchronous satelites, but that there can be geosynchronous satelites that are not geostationary. For instance, the satelites running the Sirius satelite radio service in North America are supposedly geosynchronous, but they are not geostationary. Emerymat 00:49, 14 March 2006 (UTC)
