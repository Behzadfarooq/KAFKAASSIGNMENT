It depends on whether or not you use underscores on the block screen. The following links both block the same user:

  • [3]
  • [4]

But one uses plus signs, which change to spaces in rendering (use %2B for a literal "+"), and the other uses underscores. {{urlencode:}} works for template parameters, see Template:Admin, e.g., but not for mediawiki-space parameters such as $1, which are encoded as %241 before the parameter is even evaluated. Apparently brion fixed this for at least a few hours, but that change got reverted back because it broke something else. — Jun. 2, '06 [04:39] <freak|talk>

Would it be useful to change the boilerplate on MediaWiki:Blockiptext to suggest that we use underscores? It's a band-aid, but might help until a bugfix. — xaosflux Talk 04:44, 2 June 2006 (UTC)
Wouldn't be worth it, no. It's quicker (in the event that the block log link is broken) to click on the contributions page, then go to the block log from there, than it is to change spaces to underscores in a long name such as:
