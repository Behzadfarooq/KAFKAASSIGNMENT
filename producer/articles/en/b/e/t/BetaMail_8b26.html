[edit] Internet Based Beta Mail Applications

This page is intended to compare and contrast the major consumer oriented internet based beta email providers from a product design / feature soundness perspective.

Refer to Comparison of web based email providers for a factual comparison of the haves and have nots.

Three dominant products / services which are in beta are:

  • Gmail
  • Window Live Mail
  • Yahoo Mail

Both Yahoo and Windows Live Mail have their fall-back legacy(so to speak) or classic interfaces which the users can switch back to at any time.

Both Yahoo Mail (Beta) and Windows Live Mail have taken a conventional approach to email where they've resorted to using conventional fat client like interfaces, and interactions.

GMail on the other hand has predominantly put its money on a conversational style of email organization.

Both Yahoo and Windows Live Mail have been playing catch-up in this arena, ever since GMail came out.

All products employ asynchronous javascript (AJAX) type interactions on the user interface layer and for communicating with the server.

[edit] Problems and Plus Points with GMail

[edit] Problems

  • The search but not sort and the conversation only-always approach to email hinders the work flow whenever there is a need to see information from a specific user in a certain order.
  • The conversation style of email listing, while providing context and good platform to market keyword based advertisements creates confusion when having a conversation by interleaving messages when multiple players are involved.
  • The application attempts to create a discussion / message board type trail / history in a discounted way.

[edit] Plus points

  • The search indexing is extremely efficient
  • The user can wild card on text which are parts of a word
  • Integration of chat within the email application
