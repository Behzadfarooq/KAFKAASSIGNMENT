[edit] Early days

I was born 1971 in Graz, Austria and started programming as a hobby in 1986 using an MSX home computer in BASIC and machine language. By 1990, I upgraded to personal computers, where I programmed using Quick Basic and Turbo Pascal. After taking a course for C and studying man pages and books on SQL and Unix, I switched from working as an industrial electrician and instrumentation technologist for maintaining process control and feedback control systems in a paper mill to programming for a living.

[edit] UNIX Programming and systems administration

My first programming work was done in the early 90s on UNIX-Servers (SCO UNIX release 3.2 version 4.2 and HP-UX 9.04) to create SQL database applications at local companies. I became also proficient in UNIX system and network administration.

My first production deployment of Free Software was the installation of Samba on UNIX servers, obsoleting the requirement to use PC-NFS software on the clients and thereby curing and printing problems which were caused by bugs in the PC software.

[edit] Initial use of Linux

I replaced a modem terminal which was running on DOS with a Linux-based dial-on demand router which handled many dial-out and call-back calls in parallel and later configured it to also provide dial-up access to the Internet.

I also used Linux to connect two company buildings using Ethernet bridging (for NT authentication) which I converted to a router later. I also did a bit of prototype programming in Java, but in the same year (1998), I moved to Germany and started to work for SuSE to work on improving their Linux distribution.

[edit] Time at SuSE

Like everybody, I had to do 6 months of customer support and then I started with maintenance of RPM packages. After becoming the maintainer of pppd, I became responsible for dial-up (execpt for i4l).

By 2000, thanks to my integration of wvdial, dial-up was working smoothly and I was looking for new challenges, but not for a long time: At the same time, IBM released Linux for S/390, and since I was becoming curious of the technology of IBM mainframes, I took the challenge to build up a the platform of SuSE Linux for big iron.

I was the principal developer of the project and Joachim Schroeder (who recently moved to Red Hat) worked with me on the setup of the machines. This distribution became the first version of SUSE Linux Enterprise Server (SLES). After some involvement in the software patent debate in Europe, I wanted to work with more common hardware again, and I'm now working on kernel-related tasks for AMD which incude for example kernel debugging over FireWire.

[edit] Kernel contributions

By 1995, time was ready for me to make some contributions to the Linux-Kernel (only contains what can be publically described. E.g. for the work done on Linux on zSeries, I could only reference changelog entries publically):

  • Announcement of a prototype of a new linux kernel configuration sheme which introduced menus into kernel configuration. It was adopted for the Linux kernel with the derivative follow-up tools menuconfig and xconfig in which I was also involved and for which I provided fixes and updates as well.

1997:

  • I ported the driver for DigiBoard microprocessor-driven multiport serial cards to the virtual memory mapping sceme of Linux-2.1 and implemented the use of the driver as a kernel module with module parameters for configuration. For the first time, the driver could be (re)configured without a reboot.
  • Simple patch to fix undefined symbols in Linux-2.1.30
  • Fix for the time keeping in SMP kernels 2.1.34 to 2.1.41 in APIC emulation mode which brought pre-emtive scheduling back to life in these kernels.

1999:

  • While not being a kernel contribution, it's just nice to see that I had a Gigabit Ethernet running in April 1999 (yes, we had got to use it at SuSE at that time already! - The following year, our internal backbones and servers were connected using 1000mbps glass fiber) and we used ExtremeWare switches with two (at that time) Gbit Ethernet ports, at first one, then two coupled together and ended up using ACEnic GbitE cards
  • fix oops because of overloaded module
  • patch for reinstating the old buffer code of 2.2.0pre3 into 2.2.10 - workaround for a OOM deadlock
  • fixed a swap space computation error, adopted with 2.2.10ac11

2002:

  • Fixed a IPC shm race (adopted with Linux 2.4.21pre3-ac3)

2003:

  • With 2.4.20, a security hole was fixed in the ptrace system call, but the initial fix had side effects: It prevented access to /proc/<pid>/cmdline for some tasks and prevented even root from tracing programs with were started with set-user-id. For the release of 2.4.21, Marcelo Tosatti needed a fix for these side effects which he merged with v2.4.21-rc2. It was the most prominent fix in this release because of the stakes in both security and debugging ability are high and it was needed to find a balance which does not compromise one of the other side, especially not by breaking backwards compatibility for user-space while also not compromising security.
    • Thread with an initial description and inclusion of a fix which does not have the most prominent side effects
    • Mail in which I describe how the ptrace security issue is fixed correctly

2004:

  • Fixed a Cardbus card memory assignment problem for 64-bit machines

2006:

  • patch to enable pci=assign-busses on certain Notebooks which need this set to be able to detect Cardbus cards.
  • Description of my patch which generically fixes this issue for all machines which are affected
    • Official submission of the patch by Dominik Brodowski, the responsible maintainer
  • I started work on Linux kernel debugging over FireWire and gave a workshop at LinuxTag. The workshop announcement also includes my speaker biography.
