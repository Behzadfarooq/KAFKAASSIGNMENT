Bugs, Repairs, and Internal Operational News

By Mets501, 2 April 2007

Please keep in mind that features listed here may not be live on the English Wikipedia yet. Also, in general, features that do not affect the English Wikipedia, such as new extensions that are not enabled here, are not listed.

This page covers revisions up to r20946.

New features

  • Special:Linksearch can now be used with any prefix (https://, ftp://, irc://, news://, mailto://, etc). (bug 8324; r20561)
  • Special:Protectedpages can now be sorted by namespace and protection type/level, and a minimum page length can be specified. (bug 9326; r20571, r20591)
  • Special:Whatlinkshere can now be sorted by namespace. (bug 4624; r20616, r20627)
  • An RSS feed for each database dump in each language has been created. It will be linked from the summary pages soon, and will be located in the "latest" subdirectory of each dump with the name *-rss.xml. For example, the RSS feed for the English Wikipedia's enwiki-latest-pages-articles.xml.bz2 dump will be located at http://download.wikimedia.org/enwiki/latest/enwiki-latest-pages-articles.xml.bz2-rss.xml. (r20710)
  • Two new MediaWiki: messages, MediaWiki:Sp-contributions-footer and MediaWiki:Sp-contributions-footer-anon have been created as a way to place (optionally different) footer text on the Special:Contributions page for anonymous users and logged-in users, similar to the way special text is placed at the bottom of IP talk pages. (bug 9397; r20720)
  • To help with javascript and other programming, a new javascript variable wgAction has been added. The value of this variable is the action from the page's URL, defaulting to "view" when simply viewing an article. (bug 8421; r20754)
  • Special pages dedicated to images (Special:Mostimages, Special:Uncategorizedimages, and Special:Unusedimages as of now) now display image galleries instead of image lists, as is done in categories. (bug 9069; r20788)
  • A huge amount of work was done on Flagged Revisions and ConfirmEdit, and a new extension, Minimum Name Lngth was created, although neither are enabled on the English Wikipedia.
  • The "mediawikiwiki:" interwiki prefix can now be used to link to http://www.mediawiki.org. (bug 9440; r20825)

Fixed bugs

  • Fatal error no longer appears when entering an invalid name into Special:Listusers. (r20555)
  • Users are now warned when uploading an image with an uppercase extension if one with the same lowercase extension exists or if uploading a scaled down version of an existing image. (bug 3069; r20563)
  • The block log will now show expanded text instead of abbreviations for the block flags (i.e. "anonymous users only" instead of "anononly", "account creation disabled" instead of "nocreate", etc) (r20583)
  • If a page is both normally protected and cascade protected, the regular protection message will be shown when attempting to edit the page. (bug 9401; r20646)
  • Special:Recentchangeslinked now displays a redlink for nonexistent target pages, instead of silently redirecting to the nonexistent page. (bug 5817; r20807)
  • {{anchorencode:}} no longer transforms colons. (bug 8914; r20819)
  • Special:Import now gracefully skips invalid titles with a warning. (bug 2979; r20828)
  • Special:Renameuser now doesn't allow renaming a user to an invalid username, and does allow renaming a user with an invalid username to a valid one. (bug 4939; r20841)
  • The failed rollback message now parses the edit summary of the most recent edit, instead of displaying raw wikicode. (bug 9466; r20875)
  • The "My contributions" link is now bold when viewing your own contributions. (bug 4764; r20902)

Internationalization

Internationalization help is always appreciated! See m:Localization statistics for how complete the translations of languages you know are, and post any updates to bugzilla.
